Tutorials : 2. Hidden Index and Order By

  • index_hidden : [array...int]
  • order_by : [int (index), "desc" OR "asc"]
Let's say that you have 6 th for thead ["ID", "Name", "Grade", "Gender", "Birth", "Place"] or 6 array of data and you want to hide for certain index, so
  • ID : 0
  • Name : 1
  • Grade : 2
  • Status : 3
  • Gender : 4
  • Birth : 5
  • Place : 6
Just set where you want to hide the index of data.
For example, below is to hidden the index of number 0 and then the index of 0 will be hide (the data is still exist)
Or if you want to hidden some index, then hidden_index : [0,1,2]
But, the index where you already set to hidden, still can be access for parameters
Order By:
Set where you want to "Order By" first when you firstly load the data, for example order_by : [2, "desc"] where the index of 2 is "Grade" 
$('#dt-1').table({
    json: "Scripts/json.json",
    thead: ["ID", "Name", "Grade", "Gender", "Birth", "Place"],
    hide_index: [0] // You can hide multiple index [0,1,2]
    order_by: [2, "desc"]
});

Tutorials : 3. Search

  • search :
    • index
    • type : STRING ("input" OR "select")
    • placeholder : STRING (for "input" type)
    • value : ARRAY => STRING, STRING (for "select" type)
    • group_by : (for "select" type)
      • key : INT
      • value : INT
      • default: ARRAY => STRING, STRING
Index start from zero (0) even if you hide the index of 0 
search : 
[
    { index: 0, type: "input", placeholder: "Search ID..."}, // If you hide this index, this is will not show at all.
    { index: 1, type: "input", placeholder: "Search Name..."},
    { index: 2, type: "input", placeholder: "Search Grade..."},
    { index: 3, type: "select", value: [
                                    ["", "All"], 
                                    [1, "Pass"],
                                    [0, "Not Pass"],
                                ]
    }
]

Tutorials : 6. Action(s)

  • action :
    • cls : STRING
    • url : STRING
    • params : ARRAY => INT
    • text : STRING
    • isRemove : BOOLEAN (remove data/tr)
    • params : ARRAY => INT
    • field : INT (for {i} message)
    • msg : STRING
      • {i} : index from field (thead text)
      • {m} : index from td
    • changeData : Object
      • changeIndex : ARRAY => INT (index of td)
action :
[
    {   
        cls     : "edit",
        url     : "edit/",
        text    : "_edit_",
        params  : [0, 1]
    },
    {
        cls     : "delete",
        url     : "Scripts/delete.php",
        text    : "_hapus_",
        isRemove: true,
        params  : [0],
        field   : 0,
        msg     : "Do you really want to delete this {i} : {m}?"
    },
    {
        cls     : "change",
        url     : "Scripts/change.php",
        text    : "_change_",
        params  : [0],
        changeData: 
        {
            changeIndex : [1, 2] // merubah data text pada index td
        }
    }
]

/*

    
Change

    Let's say, you have change.php and then the output is:
    $json = [ "My Name 7", 90];
    echo json_encode($json);
    Then, td for Name and Grade will be changed (including if you have array_info on it).
    
Delete

    Let's say, you have delete.php and then the output is:
    $json["result"] = true;
    echo json_encode($json);
    Then, tr and the data will be removed.
    NOTE : That the "result" of json( whatever is false or true), it will true except it have an error from the server
    
NOTE

    - That my script can't modify for your json file (Except you have backend to changing it).
    - You need to run XAMPP or WAMPP, etc. Because delete.php and change.php need PhpMyAdmin

*/
